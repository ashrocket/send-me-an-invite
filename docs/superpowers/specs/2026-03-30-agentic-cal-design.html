1. Product Overview

Agentic Cal is an open-source, self-hosted scheduling platform that is AI-native. Humans book through a pretty web UI. AI agents book through MCP. Hosts customize everything — including the visual theme — through their own agent.

Audiences

  • Hosts — people who want their own scheduling page (freelancers, founders, recruiters, consultants, coaches)
  • Bookers — humans visiting the web UI or AI agents calling the MCP

What makes it different from Calendly

  • Open source, self-hostable
  • MCP server for agent-to-agent scheduling
  • Themes are prompts — tell your agent "make it brutalist" and it generates the theme
  • No vendor lock-in — hosting plugin system supports Cloudflare, Vercel, Heroku, etc.
  • Customer-friendly billing with no surprise charges

2. Architecture

Shared core, dual interface. The core library is pure functions with no HTTP or framework dependency. Both the REST API and MCP server are thin wrappers.

┌─────────────────────────────────────────────┐ │ Core Library │ │ lib/calendar.js (pure logic) │ │ check_availability · book · cancel · etc. │ │ lib/theme.js (CSS generator) │ └──────────────┬──────────────┬───────────────┘ │ │ ┌────────▼───┐ ┌─────▼──────┐ │ REST API │ │ MCP Server │ │ /api/* │ │ tools │ └────────┬───┘ └─────┬──────┘ │ │ ┌────────▼───┐ ┌─────▼──────┐ │ Web UI │ │ AI Agents │ │ (humans) │ │ (Claude, │ │ │ │ GPT, etc) │ └────────────┘ └────────────┘
  • Core library — Pure functions, no HTTP, no framework. Testable in isolation.
  • REST API — Thin wrapper for the web UI. Runs on Cloudflare Workers (or other providers via plugins).
  • MCP Server — Thin wrapper exposing the same functions as MCP tools. Published as an npm package.
  • Web UI — Static HTML/CSS/JS calling the REST API. Themed via CSS custom properties from theme.json.

3. MCP Tools

ToolDescription
check_availabilityGet open slots for a date range
book_meetingCreate a booking (returns confirmation + magic link)
cancel_meetingCancel by booking ID or magic link token
reschedule_meetingMove a booking to a new slot
get_meeting_typesList available meeting types and durations
send_inviteShare MCP endpoint URL so another agent can book
update_themeApply a theme.json to change the site's look
set_availabilityUpdate available hours, days, buffer time

Agent-to-agent flow

  1. Host's agent shares MCP endpoint: “Book time with me at agenticcal.com/ashley”
  2. Booker's agent calls check_availability with a date range
  3. Booker's agent calls book_meeting with chosen slot + booker details
  4. Both sides get email notifications
  5. Booker receives magic link + booking code for self-service management

Custom themes

A theme is a prompt. User tells their agent “make my calendar look like a 90s geocities page” and the agent generates a theme.json. No CSS knowledge required. The update_theme MCP tool applies it.

The 10 bundled themes ship as both the prompt that generated them and the resulting theme.json — so they work instantly without an API key, and users can see how each theme was described and remix them.

5. Time Slot Presentation

Smart grouping rules

  • If 3 or fewer available slots: show as a flat list (no grouping)
  • If 4+ slots: group into Morning and Afternoon sections
  • Top-of-hour slots (:00) are visually preferred (bold) over half-hour slots
  • Lunch and existing meetings are excluded from availability
  • 15-minute buffer between meetings (configurable)
Conversational AI booking (v2) Future feature: an AI chat interface alongside the calendar wizard. Requires an Anthropic or OpenAI API key. Deferred to v2.

6. Booking Lifecycle

Book → Confirmation email (both parties) ├── Magic link (booker can manage) └── Booking code (e.g. AC-7K2F) │ ├── Reschedule → new slot selection → updated emails ├── Cancel → cancellation emails └── Magic link recovery → enter email → get link resent

Emails sent

EventTo HostTo Booker
Booking confirmedYesYes (includes magic link + booking code)
Booking rescheduledYesYes
Booking cancelledYesYes
Magic link recoveryNoYes

Magic link + booking code

  • Magic link in the email: agenticcal.com/manage?token=abc123
  • Booking code on confirmation screen: AC-7K2F
  • Either can be used to manage the booking
  • “Forgot your link?” flow: enter email, get magic link resent

Email delivery

Default email provider is Resend (free tier: 3,000 emails/month, covers most hosted users). Self-hosters can bring their own SMTP/Resend/SendGrid API key for custom sender domain.

7. Hosting & Deployment

Two tiers

TierPriceWhat you get
Self-hostedFreeDeploy to your own Cloudflare account (or other provider via plugin)
Hosted$3/year or $1.50/quarteryourname.agenticcal.com — we host it, zero setup

Hosted tier architecture

  • Wildcard DNS: *.agenticcal.com points to a single Cloudflare Worker
  • Worker reads subdomain from Host header, looks up user config in D1
  • One deployment serves all hosted users
  • Wildcard SSL is free and auto-provisioned by Cloudflare

Hosting plugins

providers/ cloudflare/ ← ships with v1 adapter.js ← wires core lib to Workers runtime deploy-button.md ← one-click deploy instructions schema.sql ← D1 migration vercel/ ← community PR adapter.js ...

Each provider plugin implements: storage (read/write bookings), secrets (API keys), and email (send notifications). Anyone can submit a PR to add a new provider.

When a limit is hit, the calendar shows “This calendar is temporarily at capacity.” No overage charges, ever.

Anti-abuse

  • Rate limiting: 60 requests/min per subdomain
  • Bot protection: Cloudflare Turnstile on booking form (free)
  • Email verification required to create hosted account
  • Subdomain squatting prevention: no bookings in 90 days = dormant

Billing rules

No auto-rebill without usage or acknowledgment. Zero activity? We pause billing and email you. Active but no “I know” flag? We send 3 reminder emails (14, 7, and 1 day before renewal). Reply STOP anytime to cancel. The “I know” flag skips reminders — toggleable in settings.

9. Onboarding Wizard

Hosted at agenticcal.com/setup:

  1. Welcome “Your calendar, your agent, your rules.” Shows what you'll get.
  2. Connect Google Calendar OAuth button → authorize → pick which calendars to check for availability.
  3. Your Profile Name, headline, meeting types, available hours.
  4. Choose a Theme Pick from 10 bundled themes, or “I'll customize later with my agent.”
  5. Host or Self-Host Hosted ($3/year or $1.50/quarter) vs. self-host (pick provider, paste API key, deploy button).
  6. Done Your site is live. Here's your URL. Here's your MCP endpoint. Here's how to share it.

Each step has an “I don't have this yet” path with clear guidance.

11. v1 Scope

v1 In scope

  • Web UI: full booking flow (meeting type, date, time, details, confirmation)
  • MCP server: check_availability, book_meeting, cancel, reschedule, get_meeting_types, send_invite
  • Theme system: 10 bundled themes, custom theme via update_theme MCP tool
  • Booking lifecycle: email notifications, magic links, booking codes, reschedule, cancel

  • Onboarding wizard at agenticcal.com/setup
  • Billing: Stripe integration, $3/year or $1.50/quarter
  • Mock calendar system for development
  • Google Calendar integration (real freebusy + event creation)

v2 Deferred

  • Conversational AI booking (chat interface with Anthropic/OpenAI API key)
  • Additional hosting providers (Vercel, Heroku, Railway — via community PRs)
  • Custom domain support for hosted tier (BYOD with CNAME)
  • Multiple calendar provider support (Outlook, iCloud)
  • Recurring meeting types
  • Team/org scheduling (multiple hosts)
